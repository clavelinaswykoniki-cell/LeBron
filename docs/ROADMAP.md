## v3.0 候选方向（非承诺）

### 方向 A：转型短视频博主弹药库

**定位**：B 端（不再面向终端球迷）

**卖点**：
- 5 分钟生成一期对线视频脚本
- 输入话题 → 输出口播稿 / 配图清单 / 数据卡 / 反驳点
- 适配抖音 / B 站 / 视频号格式

**目标用户**：篮球解说 KOL / 评论员 / 二创创作者

**模仿对象**：
- [scriptsetc.com](https://scriptsetc.com) - 短视频脚本生成
- [smashvideos.io](https://smashvideos.io) - AI 视频脚本

**风险**：
- 内容创作者付费意愿待验证
- 篮球解说 KOL 总盘子小（中文 < 1 万号）
- 头部 KOL 自有团队，不会买 SaaS

**前置验证**：
- 找 3-5 个 1-10 万粉博主访谈付费意愿
- 试做 1 期脚本送给博主用，看转化

---

### 方向 B：辩论训练平台（脱离 LeBron）

**定位**：教育产品 / 工具

**卖点**：
- 扩展到任何对立话题
  - 电动 vs 燃油车
  - iPhone vs Android
  - 美剧 vs 日剧
  - 文科 vs 理科
- 用户提案话题 → 系统出双方观点结构
- 提供反驳套路 / 论据库 / 计时器

**目标用户**：
- 高考辩论队
- 大学辩论社
- 法考 / 公考备战
- 商务谈判训练

**模仿对象**：
- [Kialo](https://www.kialo.com) - 结构化辩论平台
- [debatepedia.org](http://www.debatepedia.org) - 辩论百科

**风险**：
- 教育产品 GTM 链路长（试用 → 学校采购 → 续费 18+ 个月）
- 中国辩论社市场极度碎片
- 国外有 Kialo 抢跑

**前置验证**：
- 找 2-3 所 985 辩论社 PoC
- 评估 toC 自来水增长是否成立

---

### 方向 C：双视角观点聚合器

**定位**：中性化媒体属性产品

**卖点**：
- 任何争议事件 → 同时展示正反两方观点
- 数据 / 论据 / 反驳 / 来源全部并列
- 用户读完两边再下判断

**目标用户**：
- 想了解争议事件全貌的人（媒体素养群体）
- 学生 / 研究者
- 内容创作者素材库

**模仿对象**：
- [AllSides](https://www.allsides.com) - 美国新闻双视角聚合
- [Ground News](https://ground.news) - 媒体偏向标记

**风险**：
- 内容审核挑战大（任何热点都可能涉敏感）
- 编辑成本高（人工筛观点 / 防偏见）
- 商业化路径不清（订阅 / 广告 / 数据）

**前置验证**：
- 选 5 个低敏感话题（消费品 / 影视 / 体育）做 MVP
- 看自然流量留存
- 评估编辑团队成本

## 已明确不做的方向

以下方向已评估并排除，不再考虑：

- ❌ **涉政 / 涉黄 / 涉敏感词的对立话题**
  - 审核风险不可承受
  - 用户群虽大但产品生命周期短
- ❌ **重构成 React / Vue**
  - 当前原生小程序简洁性是核心优势
  - 重构 = 1-2 个月零产出
  - 收益不明确
- ❌ **接付费 SaaS**
  - 练手项目 monetize 噪音大
  - 会扭曲产品方向（向付费意愿倾斜）
  - 个人开发者无客服 / 售后能力
- ❌ **改成游戏化为主**
  - 偏离辩论工具核心
  - 段位 / 勋章是辅助娱乐组件，不能反客为主
  - 游戏化做透要美术 / 关卡设计 / 数值，单人做不动

---

## 决策原则

1. **先验证再开发** - 任何新方向先做 1 周 PoC 不要直接全押
2. **小步快跑** - v2.x 增量发布，单次改动 < 1 周可完成
3. **真实需求驱动** - 用户反馈 ≥ 3 次同类才进 backlog
4. **保留退出能力** - 任何方向有止损线，不沉没成本
5. **honest critique 自审** - 每个 v3.0 候选方向都必须经过 devils-advocate 跑一遍再决定
