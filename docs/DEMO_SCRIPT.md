# 30 秒演示脚本

> 朋友 / 同事 / 面试官问"最近做了啥"时，按这个脚本演示。
> 目标：**30 秒内让人觉得"诶有点意思"**，而不是把所有功能讲一遍。

---

## 一句话开场（3 秒）

> "我做了个微信小程序，把抖音 / 虎扑评论区那些黑詹姆斯的话拆成反驳卡 —— 你打开看一眼。"

不用解释技术栈，不用讲架构，直接展示。

---

## 30 秒演示流程

### 第 1 秒 - 5 秒：打开 + 搜索

```
打开小程序 → 输入框已有 "8 分"
→ "比如这个 '8 分释兵权'，詹黑最爱拿这个说事..."
→ 点"生成反驳"
```

**重点**：让对方看到搜索是即时的，不卡。

---

### 第 6 秒 - 12 秒：展示结果卡

```
→ "你看，结果分四层："
  - 对方话术（评论区原话）
  - 先回这句（一句怼回去）
  - 逻辑漏洞（黑粉哪错了）
  - 同标准回旋镖（用别人证伪）

→ "底下还有四种模式：短刀 / 封口 / 长拆 / 口播"
```

**重点**：让对方看到信息结构化、专业感。

---

### 第 13 秒 - 18 秒：一键复制 + 反馈

```
→ "点这个按钮一键复制" → 触发金光特效 + 震动 + Toast
→ "可以直接粘到评论区怼回去"
```

**重点**：让对方看到**爽感**（金光 + 震动），这是产品质感的体现。

---

### 第 19 秒 - 23 秒：彩蛋

```
→ 回首页 → "看到这个 23 号了吗？" → 长按
→ 进入彩蛋页 → 巨大"23"水印 + "Strive for Greatness"
→ "这是隐藏的，长按才出来"
```

**重点**：让对方记住一个**惊喜点**。彩蛋是作品集最容易让人"哇"的设计。

---

### 第 24 秒 - 30 秒：段位 + 测试

```
→ 回首页 → 段位浮条 "我的段位：白银詹蜜 · 已读 11 张 · 已复制 3 次"
→ "你阅读卡片会升段位，像游戏一样"
→ "底下还有个球迷测试，5 题判断你是几级詹蜜，可以分享结果"
→ 点 "🧠 球迷测试" → 答两题快速演示 → 结束
```

**重点**：让对方看到**完整产品感**，不是单一工具。

---

## 关键演示原则

### ✅ 这样讲（有效）
- "你看这个" + 直接展示
- "比如" + 具体例子（**8 分释兵权 / Excel 球王**）
- 用对方能秒懂的比喻（"像游戏段位一样"）
- 让对方动手点一下（沉浸感）

### ❌ 不要这样讲
- ❌ 列功能清单（"我做了 A / B / C / D / E..."）—— 无聊
- ❌ 讲技术栈（"用了 TDesign + 原生小程序"）—— 没人在乎
- ❌ 讲数据规模（"150 张卡 / 441 别名"）—— 抽象
- ❌ 长解释商业逻辑（"我想做球迷工具市场"）—— 跑题

## 不同观众的微调

### 👨‍💻 给程序员朋友

**加 2 句话**：
> "5 个功能都是用 superpowers + 并行 subagent 写的，跑了 devils-advocate critique 修 bug。19/20 PASS。"

→ 展示工程过程而不是功能。

### 👩‍💼 给 HR / 招聘

**加 2 句话**：
> "我有 README、单元测试、隐私页都齐了，代码在 GitHub 上你可以看 commit history。"

→ 展示完整度。

### 🧑 给非技术朋友 / 球迷

**加 2 句话**：
> "评论区被詹黑骂了？打开它复制一句怼回去 —— 不用脑子。"

→ 展示场景。

### 👨‍🎓 给同学 / 老师

**加 2 句话**：
> "这是个 weekend project，主要练**产品 + 工程闭环**，跑通了从想法到上线流程。"

→ 展示学习目的。

## 应对常见问题

### "这能赚钱吗？"

> "练手项目，**不指望赚钱**。但如果转方向到「短视频博主弹药库」或「辩论训练 H5」就有可能 —— 我列了 3 个候选方向在 README。"

### "上线了吗？"

> "本地能跑，体验版可以扫码试，正式发布要过审 1-3 天，没赶时间。"

### "为啥做詹姆斯不做其他球星？"

> "因为詹黑话术最多、最有结构、最适合做反驳卡示范。技术上做完全可以扩展到任何对立话题。"

### "AI 部分接了吗？"

> "**接了但默认关**。有个 AI 增强按钮走 CloudBase + DeepSeek。默认本地兜底就够好用，AI 是 nice-to-have。"

### "为啥 5 个功能能这么快做完？"

> "一个晚上 5 个 subagent 并行写不同文件 + 我自己负责接入 + devils-advocate 自动评测 —— 流程化 + 自动化。"

## 30 秒之后（如果对方还想看）

按这个顺序展示**深度**：

1. **关于页**：勋章墙 + 项目数据看板
2. **隐私页**：合规态度
3. **错误兜底**：空 query Toast / 复制失败 Toast
4. **分类筛选**：横向滚动 chip
5. **AI 增强**：点 "AI 增强" 按钮（如果有 key）演示

每一项 5-10 秒，加起来 1 分钟内讲完。

---

## 终极一句话总结

> **"练手项目 + 作品集 - 150 张反驳卡 + 5 个娱乐功能 - 一晚上做完。"**

如果连这一句都没时间讲，就只说"我做了个小程序，你看看"，然后递手机过去。

---

## 元演示（演示这个项目本身的"演示哲学"）

如果对方问"你怎么演示一个项目"：

> "**先让人看到爽点（彩蛋 + 金光），再讲结构（功能 + 数据），最后才讲技术（栈 + 工程）**。如果对方第 10 秒没'哇'，就别讲技术了。"
